Q1a — Dataset Preprocessing

Seven-step pipeline to make raw data training-ready

1
URL Transformation — All GCP paths rewritten: joshtalks-data-collection/hq_data/hi/{folder}/{file} → upload_goai/{folder}/{file}
2
Parallel Fetch — Transcription JSONs downloaded concurrently (8 workers) with retry logic. Audio URLs transformed but streamed lazily during training.
3
Audio Resampling — All WAV files normalized to 16 kHz mono (Whisper's required input format). Stereo channels averaged.
4
Segment Extraction — Used start/end timestamps from transcription JSON to slice audio precisely.
5
Duration Filter — Removed segments <0.5s (noise, breath) and >30s (Whisper's context limit). Kept 0.5–30s range.
6
Text Normalization — Unicode NFC normalization, strip extra whitespace, remove empty transcriptions, preserve Devanagari characters faithfully.
7
Stratified Train/Val Split — 90/10 split stratified by speaker_id to prevent speaker leakage into validation set.
8
Feature Extraction — 80-channel log-mel spectrogram (25ms window, 10ms hop) via Whisper's WhisperProcessor. JSONL manifest written for streaming.

Language Model Rescoring

Add a KenLM 5-gram Hindi LM trained on Wikipedia + Common Crawl. Rescore Whisper beam candidates using LM log-probability. Reduces phonetic confusions by using surrounding word context.

Numeral Post-normalization

Apply Q2's number normalizer as a post-processing step on model output before WER computation. Align digit output (25) with word reference (पच्चीस). This is not "more data" — it's a rule-based fix.

Q3a — Classification Approach

Five-layer decision cascade

1
Unicode Structural Validity — Detect illegal Devanagari sequences: double halant (््), matra before consonant carrier, halant at word start. → HIGH confidence incorrect.
2
Dictionary Lookup — Match against verified Hindi lexicon. Hit → HIGH confidence correct. Roman-script words → correct per transcription guidelines (English loanwords).
3
Morphological Rules — Valid Hindi suffix patterns: postpositions (ने, को, से), verb endings (ता, ती, गा, गी), plural markers (ों, एं). If suffix valid + structure ok → MEDIUM confidence correct.
4
Phonotactic Heuristics — Score consonant-matra distribution. Hindi has predictable Lo→Mn→Lo patterns. Score <0.3 → likely keyboard error. 0.3–0.6 → suspicious. >0.6 → acceptable.
5
Length Sanity Check — Words >25 characters likely run-on transcription errors. Words 1 character may be valid single-aksharas (conjunctions). → MEDIUM confidence incorrect.
Q3c — Low Confidence Review (45 words manually examined):
28 correct — mostly valid proper nouns and dialectal forms the system couldn't verify in its dictionary
17 wrong — system said "possibly correct" for words that were genuine misspellings
→ 62% accuracy at low confidence. The phonotactic heuristic needs a larger training corpus to distinguish rare-but-valid words from creative typos.
Q3d — Unreliable Categories
Proper Nouns / Tribal Names — "कुड़रमा", "खांड" are phonetically valid but absent from standard dictionaries. System defaults to LOW confidence correct but cannot distinguish these from misspellings of similar proper nouns.
Conversational / Dialectal Forms — "बोहोत" (variant of बहुत), "सायद" (variant of शायद) are dialectal, not spelling errors. System incorrectly flags them as incorrect because they don't match standard dictionary spellings.

Question 4

Lattice-Based WER

Replace rigid single-reference evaluation with a sequential word lattice where each position holds all valid alternatives. Consensus across models can override an erroneous reference.

Theory & Alignment Unit

Why word-level? Why lattice?

Alignment unit: WORD — Hindi uses space-delimited words with clear boundaries. Word-level alignment maps directly to human-perceived transcription errors and is the WER standard. Subword alignment would fragment morphological variants (खरीदी vs खरीदीं) into different error counts despite being valid alternatives of the same word.

Why a lattice? A flat string reference assumes exactly one correct transcription. Real speech has multiple valid representations: number words vs digits, spelling variants (anusvara vs chandrabindu), lexical synonyms, and references written by fallible humans. The lattice encodes all of these as equally valid paths through the sequence.

Construction Algorithm
1
Align each model to reference using edit-distance DP. At each aligned position, collect all model words.
2
Initialize bins from reference words. Expand each bin with known variant pairs: number↔digit, spelling variants, lexical synonyms.
3
Consensus promotion — If ≥ 3/5 models agree on a word that differs from the reference, add that word to the bin. Model consensus is trusted over a potentially erroneous human reference.
4
Compute Lattice WER via DP where a "match" at position i occurs if the hypothesis word is in any alternative of bin i (including via variant map). Insertions/deletions handled identically to standard WER.
